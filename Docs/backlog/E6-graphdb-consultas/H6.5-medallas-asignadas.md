# H6.5 — (6.b.ii) Medallas asignadas vs. no asignadas

**Épica:** [E6 — GraphDB + consultas](README.md)
**Requisito:** 6.b.ii · **Prioridad:** `[MUST]` · **Estado:** Pendiente
**Depende de:** H6.1, H5.3 · **Habilita:** H7.4

## Historia de usuario
Como sistema, quiero listar qué medallas están asignadas y cuáles aún no, para mostrar el panorama de logros.

## Contexto (de la consigna)
6.b.ii: "Las medallas asignadas y las que aún no están asignadas."

## Criterios de aceptación
- [ ] Para una persona (o para el juego), la consulta distingue medallas **obtenidas** vs. **pendientes**.
- [ ] La lista de medallas posibles sale del catálogo del juego (M1–M3); las obtenidas, de `pbl:earnedBadge` (H5.4) o del criterio del shape (H5.2).
- [ ] El conjunto pendiente = catálogo − obtenidas (incluye correctamente las no alcanzadas).

## Tareas
- [ ] Escribir la consulta con `MINUS`/`FILTER NOT EXISTS` o `OPTIONAL`.
- [ ] Probar con una persona que tiene sólo M1 (debe listar M2/M3 como pendientes).

## Notas
- Depende de la decisión de [H5.4](../E5-shapes-medallas/H5.4-materializacion.md): si se materializó `earnedBadge`, la consulta es directa; si no, replica el criterio del shape.
