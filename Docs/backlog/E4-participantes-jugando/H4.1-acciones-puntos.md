# H4.1 — Acciones → eventos de puntos

**Épica:** [E4 — Participantes jugando](README.md)
**Requisito:** 4 · **Prioridad:** `[MUST]` · **Estado:** Pendiente
**Depende de:** H2.4, H3.3 · **Habilita:** H4.2, H5.2

## Historia de usuario
Como equipo, queremos que cada acción relevante de un participante genere un evento de puntos según la regla correspondiente, para conectar el grafo de dominio con el juego.

## Contexto (del PDF)
Req. 4: "los participantes están jugando al juego definido en 3."

## Criterios de aceptación
- [ ] Cada submission/acción relevante (de E2) genera un `pbl:PointAward` según la `PointRule` aplicable (de H3.3).
- [ ] El `PointAward` referencia la acción de dominio que lo originó (trazabilidad dominio↔juego).
- [ ] Las acciones administrativas **no** generan `PointAward`.

## Tareas
- [ ] Mapear cada tipo de acción de E2 a su regla de H3.3.
- [ ] Instanciar los `PointAward` correspondientes.

## Notas de modelado
- Este es el **punto de acople** entre `cipa:` y `pbl:`; las ontologías quedan separadas (E1/E3) y se unen acá vía la capa de datos.
