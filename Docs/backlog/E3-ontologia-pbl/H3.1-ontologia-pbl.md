# H3.1 — Ontología PBL

**Épica:** [E3 — Ontología PBL + juego](README.md)
**Requisito:** 3 · **Prioridad:** `[MUST]` · **Estado:** Pendiente
**Depende de:** H0.3 · **Habilita:** H3.2, H3.3, H4.1

## Historia de usuario
Como modeladores, queremos una ontología de ludificación independiente del dominio, para superponer la capa PBL sobre cualquier actividad de recolección.

## Contexto (del PDF)
Tríada PBL: Points, Badges, Leaderboards. Motor de reglas (configuración de actividades, asignación de puntos, gestión de insignias). Estado del jugador: billetera, colección, nivel de dificultad.

## Criterios de aceptación
- [ ] Clases/propiedades para **Points**, **Badges**, **Leaderboards**, `pbl:Activity`, `pbl:PointRule`, `pbl:PointAward` (evento de puntos).
- [ ] Estado del jugador: `pbl:PlayerState` con billetera (`pbl:pointsBalance`), colección (`pbl:earnedBadge`), nivel de dificultad.
- [ ] `pbl:Game` que agrupa actividades, reglas y medallas (para "un juego particular" de 6.b.i).
- [ ] Vive en `ontology/pbl.ttl`, **independiente** del dominio (sin importar `cipa:`; el acople se hace en E4).
