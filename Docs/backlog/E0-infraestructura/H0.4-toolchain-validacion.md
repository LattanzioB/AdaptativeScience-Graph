# H0.4 — Toolchain de validación

**Épica:** [E0 — Infraestructura y arranque](README.md)
**Requisito:** habilitador (prepara Req. 5) · **Prioridad:** `[MUST]` · **Estado:** Pendiente
**Depende de:** H0.1 · **Habilita:** E5

## Historia de usuario
Como equipo, queremos validar TTL y correr SHACL fuera de GraphDB para iterar rápido sin recargar el store en cada cambio.

## Criterios de aceptación
- [ ] Se puede validar sintaxis TTL de forma reproducible.
- [ ] Se puede correr SHACL (incluido `sh:sparql`) localmente: **pyshacl** o **RDF4J ShaclSail**.
- [ ] Script/comando documentado que toma datos + shapes y reporta conformidad.

## Tareas
- [ ] Elegir validador SHACL que soporte `sh:sparql` (clave para E5).
- [ ] Crear script de validación reproducible (`shapes/validate.*`).
- [ ] Probar el script con un caso conforme y uno no conforme.

## Notas
- `sh:sparql` debe estar soportado por el motor elegido; verificarlo antes de E5. pyshacl lo soporta.
