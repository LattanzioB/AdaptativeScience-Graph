# H2.1 — Protocolo del diagrama

**Épica:** [E2 — Poblar el grafo](README.md)
**Requisito:** 2 · **Prioridad:** `[MUST]` · **Estado:** Pendiente
**Depende de:** H1.3, H1.4 · **Habilita:** H2.4

## Historia de usuario
Como equipo, queremos instanciar el protocolo exacto del diagrama del TP, para que el grafo refleje un caso real con bifurcación y reconvergencia.

## Contexto (del diagrama)
Flujo: `Inicio → Ubicación → Identificación INICIAL (cuestionario)` que bifurca en ramas (p. ej. *Muestra general/Especie identificada* · *Copa-Hoja/Flor-Fruto/Especie NO identificada* · *Estado Cazuela / cazuela vacía*), pasando por `Perímetro → Estado del árbol` y reconvergiendo en `Observaciones → Fin`.

## Criterios de aceptación
- [ ] Instanciar el protocolo completo con `cipa:Protocolo`, `cipa:tienePrimerPaso`, pasos `cipa:PasoMultimedia`/`cipa:PasoCuestionario` y pasos terminales cuando corresponda.
- [ ] Modelar los enunciados (`EnunciadoMultimedia` / `EnunciadoCuestionario`), los `TipoRespuesta` y las `OpcionRespuestaCuestionario` necesarias.
- [ ] Modelar las **transiciones** con `cipa:disparadaPor` cuando una opción del cuestionario inicial dispara cada rama.
- [ ] Las ramas **reconvergen** en `Observaciones → Fin` (DAG, no árbol estricto).
- [ ] Marcar el primer paso mediante `Protocolo.tienePrimerPaso` y representar el cierre del flujo como paso terminal.

## Tareas
- [ ] Crear individuos de `Paso` para cada nodo del diagrama.
- [ ] Crear `Transicion` + `OpcionRespuestaCuestionario` para cada arista condicional.
- [ ] Verificar el camino con `cipa:siguientePaso+`.

## Notas de modelado
- Confirmar los nombres/etiquetas de los pasos con el diagrama del PDF; el modelo soporta variantes mientras respeten la estructura de [H1.3](../E1-ontologia-dominio/H1.3-pasos.md): la definición del paso queda en el enunciado y lo respondido queda para `MuestraRespuesta`.
