# H3.3 — Definición del juego sencillo

**Épica:** [E3 — Ontología PBL + juego](README.md)
**Requisito:** 3 · **Prioridad:** `[MUST]` · **Estado:** Completado
**Depende de:** H3.1, H3.2 · **Habilita:** H4.1

## Historia de usuario
Como diseñadores del juego, queremos definir un juego concreto con puntajes diferenciados por acción, para cumplir el "juego sencillo" de la consigna.

## Contexto (del PDF)
"Diferentes puntos para diferentes acciones." "Diferenciación entre tareas que otorgan puntos (completar un protocolo complejo) y administrativas que no." "Reglas según dificultad o precisión."

## Criterios de aceptación
- [x] Acciones con **puntajes diferenciados** (completar protocolo = 50 pts, subir foto = 20 pts, geolocalizar = 10 pts).
- [x] Al menos **una tarea administrativa que NO otorga puntos** (editar perfil = 0 pts).
- [x] Reglas (`pbl:PointRule`) parametrizadas por **dificultad/precisión** (`pbl:hasDifficulty` → Alta/Media/Baja).
- [x] Un individuo `pbl:Game` que referencia estas reglas y las 3 medallas (H3.2).

## Tareas
- [x] Definir el catálogo de acciones puntuables y la no puntuable.
- [x] Instanciar `PointRule` por acción.
- [x] Crear el `Game` que agrupa todo.
