# H2.3 — Participantes

**Épica:** [E2 — Poblar el grafo](README.md)
**Requisito:** 2 · **Prioridad:** `[MUST]` · **Estado:** Pendiente
**Depende de:** H1.6 · **Habilita:** H2.4

## Historia de usuario
Como equipo, queremos crear participantes con identidad única, para atribuir las muestras y, más tarde, los puntos.

## Criterios de aceptación
- [ ] **2–3 participantes** con identidad única.
- [ ] Datos mínimos coherentes (nombre/label, id).
- [ ] La multiplicidad simultánea se garantiza en [H2.4](H2.4-submissions-geo.md): al menos **un participante aporta en 2 proyectos distintos**.

## Tareas
- [ ] Instanciar participantes.
- [ ] Verificar unicidad de identidad.

## Notas de modelado
- "Multiplicidad simultánea" del PDF se demuestra a nivel de datos (un participante con submissions en 2 proyectos), no con una relación aseverada.
