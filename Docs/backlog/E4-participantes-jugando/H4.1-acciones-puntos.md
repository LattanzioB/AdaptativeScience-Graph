# H4.1 — Acciones → eventos de puntos

**Épica:** [E4 — Participantes jugando](README.md)
**Requisito:** 4 · **Prioridad:** `[MUST]` · **Estado:** Pendiente
**Depende de:** H2.4, H3.3 · **Habilita:** H4.2, H5.2

## Historia de usuario
Como equipo, queremos que cada acción relevante de una persona genere un evento de puntos según la regla correspondiente, para conectar el grafo de dominio con el juego.

## Contexto (del PDF)
Req. 4: "los participantes están jugando al juego definido en 3."

## Criterios de aceptación
- [ ] Cada `cipa:Muestra` o `cipa:MuestraRespuesta` relevante (de E2) genera un `pbl:PointAward` según la `pbl:PointRule` aplicable (de H3.3).
- [ ] El `pbl:PointAward` referencia la acción de dominio que lo originó (muestra completa, respuesta con adjunto multimedia, respuesta de cuestionario, etc.) para mantener trazabilidad dominio→juego.
- [ ] Las acciones administrativas **no** generan `pbl:PointAward`.

## Tareas
- [ ] Mapear cada tipo de acción de E2 (`cipa:Muestra`, `cipa:valorMultimedia`, `cipa:seleccionaOpcion`, etc.) a su regla de H3.3.
- [ ] Instanciar los `pbl:PointAward` correspondientes.

## Notas de modelado
- Este es el **punto de acople** entre `cipa:` y `pbl:`; las ontologías quedan separadas (E1/E3) y se unen acá vía la capa de datos.
- Se conservan las IRIs existentes: dominio en `cipa:`, ludificación en `pbl:` y datos concretos en `cipa-data:`.
