# H1.8 — Consistencia

**Épica:** [E1 — Ontología de dominio](README.md)
**Requisito:** 1 · **Prioridad:** `[MUST]` · **Estado:** Pendiente
**Depende de:** H1.1–H1.7 · **Habilita:** E2

## Historia de usuario
Como equipo, queremos que la ontología pase el razonador sin inconsistencias y cargue limpia en GraphDB, para construir sobre una base sólida.

## Criterios de aceptación
- [ ] La ontología pasa el razonador en Protégé (HermiT/Pellet) **sin inconsistencias** ni clases insatisfacibles.
- [ ] Carga limpia en GraphDB con el ruleset elegido (sin errores de parsing ni de consistencia).
- [ ] Se documenta cualquier axioma que se haya relajado para evitar inconsistencias (p. ej. cardinalidades).

## Tareas
- [ ] Correr el razonador en Protégé y resolver insatisfacibilidades.
- [ ] Cargar `cipa-domain.ttl` en GraphDB y verificar.
