## Historia de usuario
Como equipo, queremos prefijos e IRIs consistentes para que dominio y ludificación queden claramente separados y las instancias sean legibles.

## Criterios de aceptación
- [ ] Documento de prefijos: `cipa:` (dominio), `pbl:` (ludificación), y los estándar (`sosa:`, `prov:`, `skos:`, `foaf:`, `geo:`, `sh:`).
- [ ] Patrón de IRIs para instancias (ej.: `cipa-data:project/arboles-nativos`, `cipa-data:participant/ada`).
- [ ] Convención de nombres: clases `UpperCamelCase`, propiedades `lowerCamelCase`, individuos `kebab-case`.
