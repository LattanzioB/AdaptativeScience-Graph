# H1.3 — Pasos (jerarquía de tipos)

**Épica:** [E1 — Ontología de dominio](README.md)
**Requisito:** 1 · **Prioridad:** `[MUST]` · **Estado:** Pendiente
**Depende de:** H1.2 · **Habilita:** H1.4, H1.5, H2.1

## Historia de usuario
Como modeladores, queremos una jerarquía de tipos de paso según el dato a capturar, para representar cada nodo del protocolo del diagrama.

## Contexto (del PDF + diagrama)
Pasos comunes: Geolocalización, Temporalidad, Multimedia, Cuestionario (MC simple/múltiple con bifurcación). El diagrama añade pasos de **estado/checkbox** (Estado del árbol, Estado Cazuela) y de **texto libre** (Observaciones).

## Criterios de aceptación
- [ ] `cipa:Step` (abstracta) + subclases: `GeolocationStep`, `TemporalStep`, `MultimediaStep`, `QuestionnaireStep`, `FreeTextStep`, `StateStep`.
- [ ] `QuestionnaireStep` con `cipa:hasQuestion` → `cipa:Question` → `cipa:hasOption` → `cipa:Option`, y `cipa:selectionType` (`single` / `multiple`).
- [ ] Marcadores de paso inicial y final del protocolo (`cipa:isStartStep` / `cipa:isEndStep` o clases `StartStep`/`EndStep`).
- [ ] `Step` queda **extensible** (admite nuevos tipos sin romper el modelo).

## Tareas
- [ ] Declarar la jerarquía de `Step`.
- [ ] Modelar `Question`/`Option`/`selectionType`.
- [ ] Definir los marcadores inicial/final.

## Notas de modelado
- Los tipos texto-libre (`FreeTextStep`) y estado/checkbox (`StateStep`) surgen del **diagrama**, no del listado textual; documentarlo como extensión justificada.

## Artefactos
- `ontology/cipa-domain.ttl` (Step y subclases, Question/Option)
