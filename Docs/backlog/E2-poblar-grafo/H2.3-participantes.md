# H2.3 — Personas participantes

**Épica:** [E2 — Poblar el grafo](README.md)
**Requisito:** 2 · **Prioridad:** `[MUST]` · **Estado:** Pendiente
**Depende de:** H1.6 · **Habilita:** H2.4

## Historia de usuario
Como equipo, queremos crear personas participantes con identidad única, para atribuir las muestras y, más tarde, los puntos.

## Criterios de aceptación
- [ ] **2–3 personas** con identidad única.
- [ ] Datos mínimos coherentes (nombre/label, id).
- [ ] La multiplicidad simultánea se garantiza en [H2.4](H2.4-muestras-geolocalizadas.md): al menos **una persona aporta en 2 proyectos distintos**.

## Tareas
- [ ] Instanciar personas.
- [ ] Verificar unicidad de identidad.

## Notas de modelado
- "Multiplicidad simultánea" del PDF se demuestra a nivel de datos (una persona con muestras en 2 proyectos), no con una relación aseverada.
