# H1.5 — Muestras y respuestas

**Épica:** [E1 — Ontología de dominio](README.md)
**Requisito:** 1 · **Prioridad:** `[MUST]` · **Estado:** Pendiente
**Depende de:** H1.3, H1.6 · **Habilita:** H2.4, H4.1

## Historia de usuario
Como modeladores, queremos representar cada ejecución de un protocolo como una muestra con respuestas por paso, para registrar qué persona respondió, dónde, cuándo y qué valor aportó.

## Contexto (del PDF)
"Cada vez que una persona realiza un aporte se genera una instancia de respuesta... Ubicación, Hora, Archivo, Datos." "Atribución de Datos: cada muestra queda vinculada permanentemente al usuario que la generó."

## Criterios de aceptación
- [ ] `cipa:Muestra` representa una ejecución concreta de un protocolo.
- [ ] `cipa:Muestra` se vincula con `cipa:ejecutaProtocolo`, `cipa:respondidaPor`, `cipa:fechaHora`, `cipa:registradaEn` y `cipa:tieneRespuesta`.
- [ ] `cipa:Persona` representa a quien responde o genera la muestra.
- [ ] `cipa:Ubicacion` registra `cipa:latitud` y `cipa:longitud`; puede alinearse con GeoSPARQL si se usa WKT.
- [ ] `cipa:MuestraRespuesta` registra lo ocurrido en un paso con `cipa:correspondeAPaso`, `cipa:fechaHoraRespuesta`, `cipa:valorTexto`, `cipa:seleccionaOpcion` y/o `cipa:valorMultimedia`.
- [ ] `cipa:AdjuntoMultimedia` se referencia desde `MuestraRespuesta.valorMultimedia` y no desde el enunciado del paso.

## Tareas
- [ ] Modelar `Muestra`, `MuestraRespuesta`, `Ubicacion` y `AdjuntoMultimedia`.
- [ ] Declarar la atribución de persona (`respondidaPor`) y documentar su alineación PROV en H1.7.
- [ ] Definir reglas de compatibilidad entre `MuestraRespuesta` y el tipo de `Paso`.

## Notas de modelado
- La regla clave del nuevo modelo: por cada paso recorrido se genera una `MuestraRespuesta`.
- Si la respuesta corresponde a `PasoMultimedia`, debe usar `valorMultimedia`.
- Si corresponde a `PasoCuestionario`, debe usar `valorTexto`, `seleccionaOpcion` o ambas según `TipoRespuesta`.
- Alinear `respondidaPor` con `prov:wasAttributedTo` habilita la **consulta de inferencia** de [H6.3](../E6-graphdb-consultas/H6.3-consulta-inferencias.md).
