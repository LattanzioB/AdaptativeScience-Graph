# H3.2 — Sistema incremental de 3 medallas

**Épica:** [E3 — Ontología PBL + juego](README.md)
**Requisito:** 3 · **Prioridad:** `[MUST]` · **Estado:** Completado
**Depende de:** H3.1 · **Habilita:** H5.2

## Historia de usuario
Como diseñadores del juego, queremos tres medallas con precedencia explícita, para representar un sistema incremental de logros.

## Contexto (del PDF)
"Sistema de medallas incremental de 3 medallas." "Precedencia y Dependencia: ganar una insignia puede requerir haber obtenido otras previamente."

## Criterios de aceptación
- [x] Tres medallas (M1, M2, M3) con **precedencia explícita** vía `pbl:requiresBadge` (M2 requiere M1, M3 requiere M2).
- [x] Cada medalla con su **criterio** declarado: umbral de puntos, cantidad de aportes y/o medalla previa (Aficionado=50; Explorador=100 + Aficionado; Naturalista=200 + Explorador).
- [x] El grafo de dependencias es **acíclico**.

## Tareas
- [x] Instanciar M1, M2, M3 con `requiresBadge`.
- [x] Declarar el criterio de cada medalla (parametrizado para que E5 lo lea).

## Notas de modelado
- El criterio debe quedar legible para que el `sh:sparql` de [H5.2](../E5-shapes-medallas/H5.2-shapes-medallas.md) lo traduzca a una restricción.

## Artefactos
- `ontology/pbl.ttl` o `data/juego.ttl` (M1–M3)
