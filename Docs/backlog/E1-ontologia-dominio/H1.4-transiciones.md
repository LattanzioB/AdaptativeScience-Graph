# H1.4 — Transiciones (DAG con bifurcación)

**Épica:** [E1 — Ontología de dominio](README.md)
**Requisito:** 1 · **Prioridad:** `[MUST]` · **Estado:** Completada
**Depende de:** H1.3 · **Habilita:** H2.1, H6.2

## Historia de usuario
Como modeladores, queremos representar el orden de los pasos como un grafo dirigido con bifurcaciones condicionales, para reflejar el flujo del protocolo (una opción de cuestionario puede decidir el paso siguiente).

## Contexto (del PDF + diagrama)
"Dependiendo de la selección, se puede realizar una bifurcación en los pasos siguientes." El diagrama muestra el paso inicial bifurcando en varias ramas que reconvergen antes de `Observaciones → Fin`.

## Criterios de aceptación
- [x] `cipa:Transicion` **reificada** con `cipa:desdePaso` (Paso) y `cipa:haciaPaso` (Paso).
- [x] `cipa:disparadaPor` (Transicion → OpcionRespuestaCuestionario) es opcional: se usa sólo para bifurcaciones condicionadas.
- [x] La bifurcación condicional sólo aplica saliendo de un `PasoCuestionario`; transiciones lineales no requieren `disparadaPor`.
- [x] Atajo `cipa:siguientePaso` (Paso → Paso) para property-paths en consultas (ej.: `cipa:siguientePaso+`), derivable de las transiciones.
- [x] El grafo de pasos es un **DAG** (sin ciclos), con reconvergencia permitida.

## Tareas
- [x] Modelar `Transicion` reificada y `disparadaPor`.
- [x] Definir `cipa:siguientePaso` y cómo se deriva/relaciona con `Transicion`.

## Notas de modelado
- Reificar la transición (vs. una simple `siguientePaso`) es lo que permite colgar la **condición** (`disparadaPor`) de la arista.
