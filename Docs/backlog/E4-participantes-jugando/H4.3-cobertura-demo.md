# H4.3 — Cobertura para demostración

**Épica:** [E4 — Participantes jugando](README.md)
**Requisito:** 4 (asegura demo de 5 y 6.b) · **Prioridad:** `[MUST]` · **Estado:** Pendiente
**Depende de:** H4.1 · **Habilita:** E5, H6.5, H6.6

## Historia de usuario
Como equipo, queremos que los datos garanticen los tres estados de medalla, para que las shapes y las consultas muestren resultados distintos y convincentes en el coloquio.

## Criterios de aceptación
- [ ] Un participante que **gana M1 pero no M2**.
- [ ] Un participante que **llega hasta M2** (gana M1 y M2, no M3).
- [ ] Un participante **elegible para M3** (gana las tres).
- [ ] (Deseable) Un participante que **no gana ninguna** medalla (caso de no-conformidad).
