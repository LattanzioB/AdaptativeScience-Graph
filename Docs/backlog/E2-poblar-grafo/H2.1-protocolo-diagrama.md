# H2.1 — Protocolo del diagrama

**Épica:** [E2 — Poblar el grafo](README.md)
**Requisito:** 2 · **Prioridad:** `[MUST]` · **Estado:** Pendiente
**Depende de:** H1.3, H1.4 · **Habilita:** H2.4

## Historia de usuario
Como equipo, queremos instanciar el protocolo exacto del diagrama del TP, para que el grafo refleje un caso real con bifurcación y reconvergencia.

## Contexto (del diagrama)
Flujo: `Inicio → Ubicación → Identificación INICIAL (cuestionario)` que bifurca en ramas (p. ej. *Muestra general/Especie identificada* · *Copa-Hoja/Flor-Fruto/Especie NO identificada* · *Estado Cazuela / cazuela vacía*), pasando por `Perímetro → Estado del árbol` y reconvergiendo en `Observaciones → Fin`.

## Criterios de aceptación
- [ ] Instanciar el protocolo completo con sus pasos tipados (geo, cuestionario inicial, multimedia, estado/checkbox, texto libre, fin).
- [ ] Modelar las **transiciones** con sus `triggeredBy` (la opción del cuestionario inicial que dispara cada rama).
- [ ] Las ramas **reconvergen** en `Observaciones → Fin` (DAG, no árbol estricto).
- [ ] Marcar paso inicial y final.

## Tareas
- [ ] Crear individuos de `Step` para cada nodo del diagrama.
- [ ] Crear `Transition` + `Option` para cada arista condicional.
- [ ] Verificar el camino con `cipa:next+`.

## Notas de modelado
- Confirmar los nombres/etiquetas de los pasos con el diagrama del PDF; el modelo soporta cualquier variante mientras respeten los tipos de [H1.3](../E1-ontologia-dominio/H1.3-pasos.md).
