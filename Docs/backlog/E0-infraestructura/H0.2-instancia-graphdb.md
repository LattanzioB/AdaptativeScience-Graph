# H0.2 — Instancia GraphDB

**Épica:** [E0 — Infraestructura y arranque](README.md)
**Requisito:** habilitador (prepara Req. 6) · **Prioridad:** `[MUST]` · **Estado:** Pendiente
**Depende de:** H0.1 · **Habilita:** E6

## Historia de usuario
Como equipo, queremos una instancia local de GraphDB con inferencia para cargar el grafo y ejecutar las consultas.

## Criterios de aceptación
- [ ] Repositorio GraphDB creado con un **ruleset con inferencia** (RDFS-Plus (Optimized) u OWL-Horst) — necesario para H6.3.
- [ ] Plugin **GeoSPARQL** habilitado si se usan filtros espaciales (ver H2.4 / H1.5).
- [ ] Carga de un TTL mínimo verificada (sin errores).

## Tareas
- [ ] Instalar GraphDB (Free) y documentar la versión.
- [ ] Crear el repositorio con reasoning activado.
- [ ] Cargar un TTL de humo y correr una consulta `SELECT * { ?s ?p ?o } LIMIT 10`.

## Notas
- El ruleset debe garantizar que `subClassOf`/`subPropertyOf` generen tripletas inferidas no triviales para la consulta de inferencia de [H6.3](../E6-graphdb-consultas/H6.3-consulta-inferencias.md).
