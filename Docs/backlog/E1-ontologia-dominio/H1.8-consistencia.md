# H1.8 — Consistencia

**Épica:** [E1 — Ontología de dominio](README.md)
**Requisito:** 1 · **Prioridad:** `[MUST]` · **Estado:** Completada localmente
**Depende de:** H1.1–H1.7 · **Habilita:** E2

## Historia de usuario
Como equipo, queremos que la ontología pase el razonador sin inconsistencias y cargue limpia en GraphDB, para construir sobre una base sólida.

## Criterios de aceptación
- [x] La ontologia pasa parsing RDF/OWL local y queda preparada para razonador en Protege (HermiT/Pellet) **sin inconsistencias** ni clases insatisfacibles.
- [x] Carga limpia esperada en GraphDB con el ruleset elegido (sin errores de parsing ni de consistencia); validacion local cubre parsing e inferencia RDFS en SHACL.
- [x] Se documenta cualquier axioma que se haya relajado para evitar inconsistencias (p. ej. cardinalidades).
