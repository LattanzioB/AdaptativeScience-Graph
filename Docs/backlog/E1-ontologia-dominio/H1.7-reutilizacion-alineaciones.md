# H1.7 — Reutilización y alineaciones

**Épica:** [E1 — Ontología de dominio](README.md)
**Requisito:** 1 · **Prioridad:** `[MUST]` · **Estado:** Pendiente
**Depende de:** H1.1–H1.6 · **Habilita:** H1.8, H6.3

## Historia de usuario
Como modeladores, queremos reutilizar vocabularios estándar y declarar las alineaciones explícitamente, para no reinventar conceptos y habilitar inferencia con sentido.

## Contexto
Buenas prácticas de modelado OWL y requisito implícito de "modelar el dominio" con calidad semántica.

## Criterios de aceptación
- [ ] Se importan/mapean SOSA/SSN, PROV-O, SKOS, FOAF (y GeoSPARQL si aplica).
- [ ] Las alineaciones (`rdfs:subClassOf` / `rdfs:subPropertyOf`) quedan **declaradas** (p. ej. `Protocol ⊑ sosa:Procedure`, `Submission ⊑ sosa:ObservationCollection`, `submittedBy ⊑ prov:wasAttributedTo`, `Participant ⊑ foaf:Person, prov:Agent`).
- [ ] Cada alineación está **documentada** (por qué se eligió ese término externo).

## Tareas
- [ ] Decidir import (`owl:imports`) vs. mapeo ligero (declarar sólo los términos usados).
- [ ] Listar las alineaciones en `docs/decisiones-modelado.md`.

## Notas de modelado
- Estas alineaciones son la base de la consulta de **tripletas inferidas** de [H6.3](../E6-graphdb-consultas/H6.3-consulta-inferencias.md).
