# H3.3 — Definición del juego sencillo

**Épica:** [E3 — Ontología PBL + juego](README.md)
**Requisito:** 3 · **Prioridad:** `[MUST]` · **Estado:** Pendiente
**Depende de:** H3.1, H3.2 · **Habilita:** H4.1

## Historia de usuario
Como diseñadores del juego, queremos definir un juego concreto con puntajes diferenciados por acción, para cumplir el "juego sencillo" de la consigna.

## Contexto (del PDF)
"Diferentes puntos para diferentes acciones." "Diferenciación entre tareas que otorgan puntos (completar un protocolo complejo) y administrativas que no." "Reglas según dificultad o precisión."

## Criterios de aceptación
- [ ] Acciones con **puntajes diferenciados** (ej.: completar protocolo complejo = N pts, subir foto válida = M pts, geolocalizar = K pts).
- [ ] Al menos **una tarea administrativa que NO otorga puntos** (ej.: editar perfil).
- [ ] Reglas (`pbl:PointRule`) parametrizadas por **dificultad/precisión**.
- [ ] Un individuo `pbl:Game` que referencia estas reglas y las 3 medallas (H3.2).

## Tareas
- [ ] Definir el catálogo de acciones puntuables y la no puntuable.
- [ ] Instanciar `PointRule` por acción.
- [ ] Crear el `Game` que agrupa todo.
