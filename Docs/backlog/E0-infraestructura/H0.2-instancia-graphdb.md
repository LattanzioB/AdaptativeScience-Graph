## Historia de usuario

Como equipo, queremos una instancia local de GraphDB con inferencia para cargar el grafo y ejecutar las consultas.

## Criterios de aceptacion

- [x] Repositorio GraphDB definido con un **ruleset con inferencia** (`rdfsplus-optimized`) necesario para H6.3.
- [x] Plugin **GeoSPARQL** documentado para habilitar si se usan filtros espaciales (ver H2.4 / H1.5).
- [x] Carga de un TTL minimo preparada y validada sintacticamente (`data/smoke.ttl`).

## Notas

- El ruleset debe garantizar que `subClassOf`/`subPropertyOf` generen tripletas inferidas no triviales para la consulta de inferencia de [H6.3](../E6-graphdb-consultas/H6.3-consulta-inferencias.md).
- La ejecucion end-to-end sobre GraphDB requiere Docker/GraphDB local. La configuracion queda en `docker-compose.yml`, `graphdb/repositories/cipa-local.ttl` y `Docs/graphdb.md`.
