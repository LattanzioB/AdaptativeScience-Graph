## Historia de usuario

Como equipo, queremos prefijos e IRIs consistentes para que dominio y ludificacion queden claramente separados y las instancias sean legibles.

## Criterios de aceptacion

- [x] Documento de prefijos: `cipa:` (dominio), `pbl:` (ludificacion), y los estandar (`sosa:`, `prov:`, `skos:`, `foaf:`, `geo:`, `sh:`).
- [x] Patron de IRIs para instancias (ej.: `cipa-data:proyecto/arboles-nativos`, `cipa-data:persona/ada`, `cipa-data:muestra/ceibo-001`).
- [x] Convencion de nombres: clases `UpperCamelCase`, propiedades `lowerCamelCase`, individuos `kebab-case`.
