# H1.4 — Transiciones (DAG con bifurcación)

**Épica:** [E1 — Ontología de dominio](README.md)
**Requisito:** 1 · **Prioridad:** `[MUST]` · **Estado:** Pendiente
**Depende de:** H1.3 · **Habilita:** H2.1, H6.2

## Historia de usuario
Como modeladores, queremos representar el orden de los pasos como un grafo dirigido con bifurcaciones condicionales, para reflejar el flujo del diagrama (la respuesta a un cuestionario decide el camino siguiente).

## Contexto (del PDF + diagrama)
"Dependiendo de la selección, se puede realizar una bifurcación en los pasos siguientes." El diagrama muestra el paso inicial bifurcando en varias ramas que reconvergen antes de `Observaciones → Fin`.

## Criterios de aceptación
- [ ] `cipa:Transition` **reificada** con `cipa:from` (Step) y `cipa:to` (Step).
- [ ] `cipa:triggeredBy` (Transition → Option): la opción que dispara la bifurcación.
- [ ] La bifurcación condicional sólo aplica saliendo de un `QuestionnaireStep`; transiciones lineales no requieren `triggeredBy`.
- [ ] Atajo `cipa:next` (Step → Step) para property-paths en consultas (ej.: `cipa:next+`), derivable de las transiciones.
- [ ] El grafo de pasos es un **DAG** (sin ciclos), con reconvergencia permitida.

## Tareas
- [ ] Modelar `Transition` reificada y `triggeredBy`.
- [ ] Definir `cipa:next` y cómo se deriva/relaciona con `Transition`.

## Notas de modelado
- Reificar la transición (vs. una simple `next`) es lo que permite colgar la **condición** (`triggeredBy`) de la arista.
